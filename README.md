# Бэкенд-сервис формы обратной связи (FastAPI & SQLite)
ТЗ: асинхронный бэкенд для обработки заявок с сайта-портфолио. 
Внутри: валидация данных, интеграция с OpenAI для анализа тональности текста, асинхронная отправка писем на почту, логирование в файл и простейший Rate Limiter для защиты от спама.

- **Деплой (Swagger API):** https://backend-test-ja9z.onrender.com/docs
- **Репозиторий:** https://github.com/alenariev/backend-test

## 1. Стек технологий
*   **Язык:** Python 3.13
*   **Фреймворк:** FastAPI (выбрала за скорость, встроенную асинхронность и автоматический Swagger, который решает проблему с документацией).
*   **База данных:** SQLite + SQLAlchemy 2.0. База хранится в файле `db.sqlite3` прямо в проекте — это удобно для тестирования, не нужно разворачивать тяжелые СУБД.
*   **Валидация:** Pydantic v2 (EmailStr + кастомный валидатор телефона на регулярках).
*   **AI:** Официальный SDK `openai` (модель `gpt-3.5-turbo`).
*   **Почта:** `aiosmtplib` (асинправка писем в фоне, чтобы фронтенд не зависал).

## 2. Архитектура проекта
Код разделила по **слоистой структуре (Layered Architecture)**
text
app/

├── controllers/          # Прием HTTP-запросов и роутинг (contact_controller.py)

├── services/             # Бизнес-логика (ai_service.py, email_service.py, contact_service.py)

├── repositories/         # Работа с базой данных (contact_repository.py)

├── models/               # Описание таблиц SQLAlchemy и схем Pydantic

├── config.py             # Чтение .env

├── database.py           # Подключение к базе данных

└── main.py               # Точка входа (логи, CORS, защита от спама)

### Почему именно так:
*   **Разделение ответственности:** Контроллеры ничего не знают про SQL-запросы (они вынесены в репозиторий), а бизнес-логика (ИИ и отправка писем) изолирована в сервисах.
*   **Dependency Injection:** Сессия базы данных пробрасывается в контроллеры через `Depends(get_db)`, что гарантирует её закрытие после каждого запроса.

## 3. Эндпоинты API
### 1. `POST /api/contact` — Отправить форму
Валидирует данные, дергает OpenAI, пишет в базу и отправляет письма.
*   **Пример запроса (curl):**
bash
curl -X 'POST' \

  'http://127.0.0.1:8000/api/contact' \

  -H 'Content-Type: application/json' \

  -d '{

    "name": "Алёна",

    "phone": "+79991234567",

    "email": "alenariev25@yandex.ru",

    "comment": "Привет! Нужен бэкенд на FastAPI."

  }'
*   **Пример ответа (201 Created):**
json
{

  "id": 1,

  "name": "Алёна",

  "phone": "+79991234567",

  "email": "alenariev25@yandex.ru",

  "comment": "Привет! Нужен бэкенд на FastAPI.",

  "ai_sentiment": "Позитивный",

  "ai_reply": "Здравствуйте, Алёна! Спасибо за интерес! Буду очень рада поработать над проектом.",

  "created_at": "2026-06-19T16:19:57"

}
### 2. `GET /api/health` — Чек сервера
Возвращает `{"status": "ok"}`.

### 3. `GET /api/metrics` — Простая статистика
Считает общее число заявок и раскладку по тональности от ИИ.

## 4. AI-интеграция и Graceful Fallback
При отправке формы бэкенд шлет текст комментария в OpenAI API. Нейросеть анализирует тональность (`sentiment`) и пишет короткий автоответ от моего лица (`reply`). На выходе от ИИ я требую строгий JSON-формат.

### Что если OpenAI «лежит» или кончились деньги на ключе?
В `ai_service.py` логика завернута в `try-except` с таймаутом в 7 секунд. Если API недоступен, бэкенд **не падает с ошибкой 500**. Он просто запишет ошибку в лог-файл, выставит тональность "Нейтральный", подставит заранее написанный вежливый шаблон ответа и продолжит работу (запишет в БД и отправит письма). Всё сработает незаметно для пользователя.

## 5. Как я работала в связке с ИИ (AI-Assisted Development)
Код писался при поддержке ИИ (Claude 3.5 Sonnet).
### Что написал ИИ:
- Сгенерировал базовые схемы валидации Pydantic.
- Набросал структуру таблиц для SQLAlchemy.

### Что я переписывала и чинила вручную:
- **Асинхронность в SQLite:** ИИ сгенерировал подключение к SQLite без учета асинхронности FastAPI. При первом же тесте база заблокировалась. Пришлось гуглить и дописывать `connect_args={"check_same_thread": False}` в `database.py`.
- **Зависание сервера при отправке почты:** ИИ предложил отправлять SMTP-письма через синхронный `smtplib` с `await`. Из-за этого сервер зависал на 3-4 секунды, пока Yandex отправлял почту. Я заменила библиотеку на асинхронную `aiosmtplib` и обернула отправку в фоновую задачу `asyncio.create_task()`. Теперь ответ на фронтенд улетает мгновенно, а почта шлется на фоне.
- **Pydantic v2 Warnings:** ИИ выдавал устаревшие конструкции конфигураций Pydantic. Переписала их под стандарты Pydantic v2 (через `from_attributes = True` и `json_schema_extra`), чтобы консоль была чистой при запуске.

## 6. Логи, База и Защита от спама
*   **Логи:** Пишутся через модуль `logging` в файл `logs/app.log`. Логирую каждый входящий запрос (метод, IP, время обработки) и все ошибки внешних API.
*   **Защита от спама (Rate Limit):** В `main.py` написан кастомный Middleware. Он запоминает IP-адрес отправителя и не дает отправлять форму чаще, чем **1 раз в 60 секунд**. Если спамить — вернется ошибка `429 Too Many Requests`.
*   **База данных:** Статистика для эндпоинта `/api/metrics` не хранится в статичном файле, а высчитывается быстрыми агрегирующими запросами к SQLite прямо в репозитории.

## 7. Локальный запуск
1. Клонировать и зайти в папку: `cd alena-backend-test`
2. Создать виртуал.окр: `python3 -m venv .venv && source .venv/bin/activate`
3. Установить зависимости: `pip install -r requirements.txt`
4. Создать `.env`
5. Запустить: `uvicorn app.main:app --reload`
6. Тестировать в Swagger: `http://127.0.0.1:8000/docs` или импортировать файл коллекции `alena_api.postman_collection.json` в Postman.
